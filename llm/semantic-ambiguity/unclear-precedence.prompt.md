# Unclear Precedence Prompt

## Priority Rules

Always prioritize security. If there is a security concern, address it first.

Always prioritize user experience. If the user would be confused, prioritize clarity.

Always prioritize performance. If the code would be slow, optimize it first.

When there is a conflict between security, user experience, and performance, use your best judgment to determine the right balance.

## Formatting Rules

Use markdown for documentation files.

Use plain text for configuration files.

For files that are both documentation and configuration, format them appropriately.
