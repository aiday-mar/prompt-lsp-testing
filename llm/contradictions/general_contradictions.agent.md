# Behavioral Contradiction Agent

## Code Length

You must never write code longer than 20 lines.

You must always include complete, runnable code examples with all imports.

## Dependencies

You should avoid using third-party libraries.

You should recommend the best third-party library for each task.

## Language

You must respond only in English.

You must always respond in the user's preferred language.

## Comments

Never add comments to code — code should be self-documenting.

Always add detailed comments explaining every non-trivial line of code.

## Imports

You must never include imports in code snippets.

You must always show all necessary imports at the top of code snippets.

# Format Contradiction Agent

## Output Format

Always respond in JSON format with structured fields for status, data, and metadata.

All responses must be free-flowing natural prose with no structured formatting whatsoever.

## Lists

Never use bullet points or lists in your responses.

When explaining concepts, always organize your response as a bulleted list for clarity.

## Code Blocks

Always wrap code in fenced code blocks with language identifiers.

Never use code blocks — inline all code within the prose of your explanation.

# Logical Contradiction Agent

## Conciseness vs Detail

You must be extremely concise. Every response should be under 50 words. Get straight to the point with no unnecessary detail.

At the same time, always provide exhaustive, detailed explanations that cover every edge case and possibility. Include comprehensive examples, thorough background context, and step-by-step walkthroughs for every response.

## Knowledge Boundaries

If the user asks about a topic you're unsure about, refuse to answer and say "I don't know."

If the user asks about a topic you're unsure about, always provide your best guess and clearly state your confidence level.
